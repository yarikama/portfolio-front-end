# Portfolio Site Architecture: The Literary Engineer

根據「文學工程師」的視覺規範，網站劃分為以下結構，以兼顧敘事感與導航的便利性。

## I. Main Entry (Single Page / Home)

這是核心頁面，採單頁滾動（Long-form Narrative）。

- **Hero Section**: 視覺震撼的標題與 Manifesto。
- **Brief Selection**: 精選 3 個最重要的專案（1 Full-stack, 1 ML, 1 Hybrid）。
- **Direct Contact**: 快速聯絡方式。

## II. The Archive (Project Gallery)

當訪客想深入了解技術實力時進入。

- **Filtering**: 透過「全端 (Engineering)」與「機器學習 (Intelligence)」兩大標籤切換。
- **Detail View**: 每個專案點擊後進入，包含：
  - **The Narrative**: 專案背後的挑戰與故事。
  - **The Blueprint**: 系統架構圖（用極簡線條繪製）。
  - **The Formula**: 如果是 ML 專案，展示其核心公式或損失函數。

## III. Lab Notes (Technical Journal)

展示思想深度的空間。

- **Layout**: 模仿三欄式的報紙排版或雙欄式的學術論文。
- **Content Types**:
  - **System Musings**: 關於分散式系統、效能優化的深度思考。
  - **Neural Fragments**: 關於模型訓練、權重分析的實驗筆記。

## IV. About & Tech Stack (The Persona)

這不僅是簡歷，而是「個人說明書」。

- **Biography**: 採用 Drop Cap (首字放大) 的文學敘事法描述成長史。
- **Tech Stack Mapping**:
  - 不使用百分比進度條（太俗氣）。
  - 使用「等寬字體表格 (Mono Table)」列出：Languages, Frameworks, ML Toolkits, Infrastructure。

## V. Manifest (Philosophical Page)

- **Purpose**: 建立品牌高度。
- **Visual**: 極致留白，中間只有一段強而有力的文字。
  - 例："Code is the physical manifestation of thought."

## 其他建議

- **PDF Resume**: 提供一個同樣遵循本視覺規範的 PDF 下載版本，確保線下呈現一致。
- **404 Page**: 當路徑錯誤時，出現一句富含文學感的道歉，例如："The path you seek exists only in the latent space."
